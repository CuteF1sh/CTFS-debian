# ПР №6. Анализ сетевого трафика: tcpdump и Wireshark

## 1. Разбор строки tcpdump

Строка: 17:56:57.087824 IP 127.0.0.1.42290 > 127.0.0.1.8080: Flags [S], seq 2480263338, win 65495, length 0

| Поле | Значение | Что означает |
|------|----------|--------------|
| 17:56:57.087824 | Время | Время захвата пакета |
| IP | Протокол | Используется протокол IPv4 |
| 127.0.0.1.42290 | Источник | IP и порт отправителя |
| > | Направление | Поток данных от источника к получателю |
| 127.0.0.1.8080 | Назначение | IP и порт получателя |
| Flags [S] | SYN-флаг | Начало TCP-соединения |
| length 0 | Длина | Длина полезной нагрузки (0 байт) |

## 2. Перехваченные данные HTTP

В трафике HTTP виден пароль в открытом виде: username=admin&password=supersecret123

Это опасно, потому что любой, кто имеет доступ к сети (другие пользователи Wi-Fi, администратор сети, злоумышленник) может перехватить и прочитать эти данные.

## 3. Follow TCP Stream

В окне Follow TCP Stream виден полный диалог клиент-сервер:
- Клиент отправляет POST /login с параметрами user=bob&pass=qwerty
- Сервер отвечает 501 Unsupported method

Злоумышленник, перехвативший этот трафик, видит логин, пароль и всё содержимое запроса и ответа в открытом виде.

## 4. HTTP vs HTTPS

| Что наблюдаем | HTTP (порт 8080) | HTTPS (порт 443) |
|---------------|------------------|------------------|
| IP-адреса сторон | Видны | Видны |
| Порт назначения | Виден (8080) | Виден (443) |
| Заголовки HTTP | Видны полностью | Не видны (зашифрованы) |
| Тело запроса | Видно открытым текстом | Не видно |
| Логин и пароль | Видны | Не видны |
| Имя домена | Видно в Host | Видно только в DNS |

## 5. DNS и приватность

При открытии HTTPS-сайтов DNS-запросы (порт 53) передаются в открытом виде. Это означает, что даже при использовании HTTPS злоумышленник может узнать, какие сайты вы посещаете.

## 6. Base64

Декодирование строки dmFzeWE6TX1QYXNzd29yZDIwMjQ=:
vasya:M}Password2024

Base64 не является шифрованием, это просто кодировка. Любой может декодировать строку обратно. Для защиты данных нужно использовать настоящее шифрование (TLS, AES).

## 7. tshark vs tcpdump vs Wireshark

| Инструмент | Когда использовать |
|------------|---------------------|
| tcpdump | Консольный захват, подходит для серверов и автоматизации |
| tshark | Консольный анализ, фильтрация, статистика |
| Wireshark | Графический интерфейс, удобен для детального анализа |

## 8. Каналы утечки через сеть

Меры защиты, которые закрывают сетевой канал утечки:
- Шифрование трафика (HTTPS, TLS)
- VPN для всего трафика
- DNS over HTTPS (DoH) или DNS over TLS (DoT)
- Отказ от протокола HTTP
